In the previous post we covered the **Load** step — pulling raw text out of PDFs, text files, and websites. Now we move on to **Embedding**, where things get a little more interesting.

> Quick recap: after loading, documents go through a **splitting** step that breaks them into manageable chunks. Embedding happens on those chunks.

## What is an embedding?

Computers can't understand sentences the way we do. So before we can store or search our documents intelligently, we need to translate each chunk of text into a list of numbers called a **vector**.

Two chunks that mean similar things will produce vectors that are numerically close to each other. Two chunks about completely different topics will produce vectors that are far apart. This numeric "distance" is what makes semantic search possible — instead of matching keywords, we match *meaning*.

For example:

- "The sky is blue" and "The sky appears azure" → very close vectors
- "The sky is blue" and "I enjoy pasta" → very distant vectors

## The Embed step

To generate these vectors we use a **sentence transformer model**. In the code below, we use `all-MiniLM-L6-v2`, a lightweight open-source model from HuggingFace that converts each text chunk into a vector of 384 numbers.

### Loading all PDFs from a folder

Before we embed anything, we need our documents. This helper function walks a directory, loads every PDF it finds, and tags each page with metadata so we know where it came from:

```python
from pathlib import Path
from langchain_community.document_loaders import PyPDFLoader

def process_all_pdfs(pdf_directory):
    all_documents = []
    pdf_files = list(Path(pdf_directory).glob("**/*.pdf"))
    print(f"Found {len(pdf_files)} PDF files.")

    for pdf_file in pdf_files:
        try:
            loader = PyPDFLoader(str(pdf_file))
            documents = loader.load()
            for doc in documents:
                doc.metadata["source"] = pdf_file.name
                doc.metadata["file_type"] = "pdf"
            all_documents.extend(documents)
            print(f"Loaded {len(documents)} pages from {pdf_file.name}")
        except Exception as e:
            print(f"Skipped {pdf_file.name}: {e}")

    print(f"Total pages loaded: {len(all_documents)}")
    return all_documents

all_pdf_documents = process_all_pdfs("c:/Code/RAG/data/pdf")
```

`Path.glob("**/*.pdf")` searches recursively, so nested subfolders are covered automatically. Any file that fails to load is skipped with a warning rather than crashing the whole run.

### Generating the embeddings

Now for the embedding step itself:

```python
import numpy as np
from sentence_transformers import SentenceTransformer

class EmbeddingManager:
    def __init__(self, model_name="all-MiniLM-L6-v2"):
        self.model_name = model_name
        self.model = None
        self._load_model()

    def _load_model(self):
        print(f"Loading embedding model: {self.model_name}...")
        self.model = SentenceTransformer(self.model_name)
        dim = self.model.get_sentence_embedding_dimension()
        print(f"Model loaded successfully. Embedding dimension: {dim}")

    def generate_embeddings(self, texts: list[str]) -> np.ndarray:
        if not self.model:
            raise ValueError("Model not loaded.")
        print(f"Generating embeddings for {len(texts)} documents...")
        vectors = self.model.encode(texts, show_progress_bar=True)
        print(f"Generated embeddings with shape: {vectors.shape}")
        return vectors

embedding_manager = EmbeddingManager()
```

**Output:**

```
Loading embedding model: all-MiniLM-L6-v2...
Model loaded successfully. Embedding dimension: 384
```

Each call to `generate_embeddings(["some text", "more text"])` returns a NumPy array with shape `(2, 384)` — one row per chunk, 384 columns per row. Each column is a dimension in the model's "meaning space".

### A note on the warnings

When you run this for the first time, you may see a couple of warnings — both are harmless:

- **Unauthenticated HF Hub request** — the model downloads fine without a token. Setting a `HF_TOKEN` environment variable simply unlocks higher rate limits if you're downloading many models.
- **Symlinks warning on Windows** — HuggingFace's cache system prefers symlinks for efficiency, but Windows restricts them by default. Your files will still cache correctly; you may just use slightly more disk space. To fix it properly, enable Developer Mode in Windows settings.
- **`embeddings.position_ids` unexpected key** — this is a known, benign mismatch that appears when loading sentence transformer models. It does not affect the quality of your embeddings and can safely be ignored.

### Putting it all together

```python
# Extract the raw text from each loaded page
texts = [doc.page_content for doc in all_pdf_documents]

# Generate embeddings for all chunks
vectors = embedding_manager.generate_embeddings(texts)

print(f"Embedded {vectors.shape[0]} chunks, "
      f"each represented as {vectors.shape[1]} numbers.")
```

## Why this model?

`all-MiniLM-L6-v2` is a popular starting point because it is fast, small enough to run on a laptop CPU, and produces good results for general-purpose documents. If you later need higher accuracy — for example, on technical or domain-specific text — you can swap in a larger model like `all-mpnet-base-v2`. The `EmbeddingManager` class accepts any HuggingFace sentence transformer name, so the change is a single line.
